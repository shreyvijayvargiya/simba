# Users API

Manage users through the version 2 API.

## Endpoints

### GET /users

List all users (admin only).

**Query Parameters:**

- \`page\` (optional): Page number (default: 1)
- \`limit\` (optional): Items per page (default: 10)
- \`search\` (optional): Search query

**Example:**

\`\`\`bash
GET /users?page=1&limit=20&search=john
\`\`\`

**Response:**

\`\`\`json
{
  "success": true,
  "data": {
    "users": [
      {
        "id": "123",
        "email": "user@example.com",
        "name": "John Doe",
        "role": "user",
        "createdAt": "2024-01-01T00:00:00Z"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 100,
      "totalPages": 5
    }
  }
}
\`\`\`

### GET /users/:id

Get a specific user by ID.

**Response:**

\`\`\`json
{
  "success": true,
  "data": {
    "id": "123",
    "email": "user@example.com",
    "name": "John Doe",
    "role": "user",
    "createdAt": "2024-01-01T00:00:00Z",
    "updatedAt": "2024-01-01T00:00:00Z"
  }
}
\`\`\`

### PUT /users/:id

Update a user (admin or own account).

**Request Body:**

\`\`\`json
{
  "name": "John Updated",
  "role": "admin"
}
\`\`\`

### DELETE /users/:id

Delete a user (admin only).

**Response:**

\`\`\`json
{
  "success": true,
  "message": "User deleted successfully"
}
\`\`\`

## User Roles

| Role | Permissions |
|------|------------|
| \`user\` | Basic access |
| \`admin\` | Full access |
| \`moderator\` | Content moderation |
